# Using TypeScript with the Sanity SDK

The Sanity SDK is fully typed and supports explicit type parameters on all data-fetching hooks. You can define your own TypeScript interfaces or use types generated by your Sanity Studio's typegen tooling, then pass them as type parameters to SDK hooks for full type safety.

## Defining Your Types

You can define types for your documents manually or import them from a generated types file (e.g., one produced by running `sanity typegen generate` in your Studio project).

### Manual Type Definitions

```typescript
import {type SanityDocument} from '@sanity/sdk-react'

interface Article extends SanityDocument {
  _type: 'article'
  title: string
  body: string
  publishedAt: string
}

interface Author extends SanityDocument {
  _type: 'author'
  name: string
  role?: string
  awards?: string[]
}
```

### Using Studio-Generated Types

If your Sanity Studio project uses `sanity typegen generate`, you can import the generated types into your SDK application:

```typescript
import type {Article, Author} from './path/to/sanity.types'
```

## Using Types with Hooks

### `useDocument`

Pass your document type as a type parameter to get fully typed document data:

```tsx
import {useDocument, type DocumentHandle} from '@sanity/sdk-react'

interface Book extends SanityDocument {
  _type: 'book'
  title: string
  author: string
}

function BookView({doc}: {doc: DocumentHandle}) {
  const {data: book} = useDocument<Book>({...doc})
  return (
    <h1>
      {book?.title ?? 'Untitled'} by {book?.author ?? 'Unknown'}
    </h1>
  )
}
```

You can also type path selections:

```tsx
function BookTitle({doc}: {doc: DocumentHandle}) {
  const {data: title} = useDocument<string>({...doc, path: 'title'})
  return <h1>{title ?? 'Untitled'}</h1>
}
```

### `useEditDocument`

```tsx
import {useEditDocument, type DocumentHandle} from '@sanity/sdk-react'

function BookEditor({doc}: {doc: DocumentHandle}) {
  const editBook = useEditDocument<Book>({...doc})

  const handleTitleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    editBook((prev) => ({...prev, title: e.target.value}))
  }

  return <input onChange={handleTitleChange} />
}
```

### `useQuery`

```tsx
import {useQuery} from '@sanity/sdk-react'

interface MovieResult {
  _id: string
  title: string
  releaseDate: string
}

function MovieList() {
  const {data} = useQuery<MovieResult[]>({
    query: '*[_type == "movie"]{_id, title, releaseDate}',
  })

  return (
    <ul>
      {data?.map((movie) => (
        <li key={movie._id}>{movie.title}</li>
      ))}
    </ul>
  )
}
```

### `useDocumentProjection`

```tsx
import {useDocumentProjection, type DocumentHandle} from '@sanity/sdk-react'

interface AuthorSummary {
  name?: string
  bookCount?: number
}

function AuthorCard({doc}: {doc: DocumentHandle}) {
  const {data} = useDocumentProjection<AuthorSummary>({
    ...doc,
    projection: `{
      name,
      "bookCount": count(favoriteBooks)
    }`,
  })

  return (
    <div>
      <h3>{data.name}</h3>
      <p>{data.bookCount} books</p>
    </div>
  )
}
```

## Document Handles

`DocumentHandle` is a generic type that accepts type parameters for the document type, dataset, and project ID. You can use these to narrow the handle to a specific document type:

```typescript
import {type DocumentHandle} from '@sanity/sdk-react'

function BookComponent({doc}: {doc: DocumentHandle<'book'>}) {
  // doc.documentType is narrowed to 'book'
}
```

The SDK provides helper functions like `createDocumentHandle` that capture literal string types automatically:

```typescript
import {createDocumentHandle} from '@sanity/sdk-react'

const handle = createDocumentHandle({
  documentId: '123',
  documentType: 'book',
  dataset: 'production',
  projectId: 'abc',
})
// handle.documentType is typed as 'book' (not string)
```

Alternatively, use `as const` on plain objects to preserve literal types:

```typescript
const handle = {
  documentId: '123',
  documentType: 'book',
  dataset: 'production',
  projectId: 'abc',
} as const
```
